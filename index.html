  • 命令行的方法大致理解了
  • 先在github上建一个新的空的仓库
  • 配置好ssh钥匙，怎么配的已然忘了，回头再总结吧，钥匙一台机器配一个就行了，以后每次push都不用管了
  • 在本地建一个文件夹作为一个本地的仓库
  • git init 产生一个.git的文档 Initialized empty Git repository in F:/project/.git/
  • 目的是在创建的文件夹里初始化一个空的git 仓库
  • 然后可以在这个文件夹里创建一些文件 html css js img 文件夹 等等
  • 然后git add +想要上传的文件 把想要提交的文件存入暂存
  • 然后git commit -m +注释信息 把暂存的东西提交
  • 第一次提交要增加一个将要提交到的那个仓库的地址 git remote add origin git@github.com:CrazyRabbitA/blog.git
  • 后续再往那个仓库提交东西就不用这一句了
  • 最后git push -u origin master
  • 刷新github就可以了
  • 本地更改完文件就不用从第一步开始了
  • 直接 git add 然后git commit 最后git push就行了
